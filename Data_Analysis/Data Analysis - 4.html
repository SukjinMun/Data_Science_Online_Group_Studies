결정계수(R-squared, R2)¶

  • coefficient of determination
  • 회귀 분석이 얼마나 잘 되었는지 확인하는 평가지표
  • 회귀에서는 예측하는 값이나 타깃이 임의의 수치이므로 정확한 숫자를 맞힌다는 것은 거의 불가능 함
  • 0~1 사이의 값을 갖고, 1에 가까울수록 예측력이 좋다고 판단(데이터를 잘 설명하는 모델)
  • 샘플의 타깃과 예측한 값의 차이를 제곱하여 더함
  • 타깃과 타깃 평균의 차이를 제곱하여 더한 값으로 나눔
  • 타깃의 평균 정도를 예측하는 수준이라면(분자와 분모가 비슷하다면) R2는 0에 가까워지고, 예측이 타깃에 가까워지면(분자가 0에 가까워지기 때문에) 1에 가까운 값이 됨

과대적합(Over-fitting)과 과소적합(Under-fitting)¶

  • Over-fitting: 샘플에 모델이 과하게 적합되어 있는 상태
  • Under-fitting: 샘플에 모델이 너무 부족하게 적합되어 있는 상태

다양한 회귀문제 평가지표들¶

  • R(root), M(Mean), S(Squard), L(Log), P(Percentile), A(Absolute), E(Error)
  • MAPE: 오차의 비율을 절대값 씌우고 평균
  • MAE: 오차를 절대값 씌워서 평균
  • MSE: 오차를 제곱해서 평균
  • RMSE: 오차를 제곱하여 평균한 것에 제곱근 적용
  • RMSLE: 로그오차를 제곱하여 평균한 것에 제곱근 적용

Feature Selection(변수 선택법)¶

  • 과적합을 방지하고, 일반화를 달성하기 위해 사용하는 방법
  • 모든 변수를 다 사용하지 않고 골라서 사용(어떤걸 넣을지, 어떤걸 뺄지)
  • 변수를 선택해서 사용하기 때문에 학습속도 증진을 기대할 수 있음(데이터 용량 감소)
  • Filter, Wrapper, Embed

Filter¶

  • 기초통계량을 기반으로 Feature 간의 관계에서 높은 상관계수(영향력)를 가지는 feature를 사용하는 방법
  • ex) 표준편차 0에 가까우면(해당 Column에 저장된 데이터들이 거의 동일) 해당 feature는 삭제
  • IOT Sensor data, Smart Factory 쪽 센서류의 경우 종종 발생

Wrapper¶

  • feature를 하나씩 넣고, 빼면서 평가지표 점수 확인(시행착오 거친다)
  • feature의 조합에 따라서 평가지표의 점수가 달라진다는 것으로부터 출발
  • feature 조합별로 평가지표 점수 확인해서 점수가 가장 좋은 조합을 찾아냄
  • Wrapper 방법: 전진 선택, 후방 제거, 단계별 선택

그룹 연산¶

  • 특정 기준으로 그룹을 분할하여 처리
  • 그룹 연산은 데이터를 집계, 변환, 필터링하는데 효율적
In [18]:
# 그룹 분할: df.groupby(기준이 되는 컬럼)
# 그룹에 적용: 그룹.agg(매핑 함수)
# 특정 컬럼 A를 정해서 A의 경우의 수로 그룹 짓고, 
# 각 그룹별로 한 그룹당 특정 컬럼 B를 정해 B 안의 값들을
# 일괄적으로 합쳐서 계산

# year 컬럼의 경우의 수 기준으로 Row들을 그룹 지음
year = df.groupby('year')

# .agg() -> 그룹 별로 합산 연산
# {'count':np.mean} -> count 컬럼에 대해 np.mean 연산을 일괄 수행
year_count = year.agg({'count':np.mean})
year_count.reset_index()
